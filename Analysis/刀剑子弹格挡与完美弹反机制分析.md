# 刀剑子弹格挡与完美弹反机制分析（留档）

## 结论摘要

- **普通格挡挡子弹**：需要满足可挡子弹前置（刀剑、远程来袭、技能达标、朝向达标）并处于格挡/偏斜状态。
- **普通反弹（Deflect）**：在格挡状态下，若解锁对应技能并且耐力超过阈值，可触发 `WasBulletDeflected`。
- **完美弹反（Parry）**：在“短时间 Deflect 窗口”内命中子弹时触发 `WasBulletParried`，属于你说的“击中瞬间格挡”机制。
- **反弹落点可控到准星附近**：仅 `IsBlocking` 触发 `WasBulletDeflected` 时，也会沿准星方向反弹；在子弹时间激活或 `WasBulletParried` 时，会进一步做“准星附近敌人组件/胸口位”的目标吸附，命中更稳定。

---

## 1) 子弹可被刀剑防御的前置条件

位置：`damageManager.swift` 的 `CanBlockBullet(hitEvent)`

核心判定：

1. 防御武器存在。
2. 防御武器是刀剑（`IsBlade()`）。
3. 来袭攻击必须是远程（`AttackData.IsRangedOnly(...)`）。
4. 玩家已解锁里程碑技能并达到阈值（`Reflexes_Right_Milestone_2 >= 2`）。
5. 朝向与视角判定通过（不是全方向）。

这一步只决定“**能不能进入刀挡子弹体系**”。

---

## 2) 命中瞬间如何分流：Block / Deflect / Parry

位置：`damageManager.swift` 的 `ProcessDefensiveState(hitEvent)`

在子弹分支里（`currentStamina > 0 && CanBlockBullet`）还会检查：

- `IsBlocking == 1` 或 `IsDeflecting == 1`。

之后分流：

- 若已购入 `Reflexes_Right_Perk_2_1` 且耐力充足：
  - `IsBlocking == 1` → 标记 `WasBulletDeflected`
  - `IsDeflecting == 1` → 标记 `WasBulletParried`
- 否则仅 `WasBulletBlocked`（普通挡住，不进入弹反）

耐力阈值公式：

`currentStamina > totalStamina * Reflexes_Right_Milestone_2_StaminaDeflectPerc`

注意是**严格大于**，不是大于等于。

---

## 3) “完美弹反窗口”来自哪里

位置：`meleeTransitions.swift`

关键状态：`MeleeDeflect`

- 进入条件：按下格挡、武器可弹反、未禁用、无冷却等。
- 该状态有短持续时间（默认参数 `duration = 0.40`，具体可被配置覆盖）。
- 进入状态时给玩家加上 `IsDeflecting = 1`。
- 退出状态时移除 `IsDeflecting`，并启动 `Deflect` 冷却。

因此，“**子弹命中瞬间按格挡触发完美弹反**”本质就是：

- 命中时刻玩家是否处于 `IsDeflecting` 的短窗口内。

---

## 4) 反弹弹道为何能靠近准星

位置：`damageSystem.swift` 的 `ProcessBulletDeflect(...)`

核心流程：

1. 先读取当前准星数据（`GetDefaultCrosshairData`）。
2. 若满足：
   - 子弹时间激活（`Reflexes_Right_Perk_2_3` + Time Dilation Active），**或**
   - 本次命中是 `WasBulletParried`（完美弹反）
3. 则寻找准星最近敌人，并将目标点尽量修正到：
   - 目标组件位置（角度足够小时），
   - 或胸口槽位，
   - 或目标世界坐标。
4. 再基于该点生成反弹攻击（`..._Deflect` / `..._ParryDeflect`）。

所以体感上会出现两层效果：

- 仅 `IsBlocking`：反弹已会“沿准星方向”走。
- `WasBulletParried` 或子弹时间：在“沿准星方向”基础上，再叠加目标吸附，表现为更像自动贴准星目标。

### 4.1 实测修正说明（与你当前 Ninja 配置一致）

你当前脚本常驻 `IsBlocking` 后，确实可在不进入 `IsDeflecting` 的前提下实现“反弹到准星方向”的体感。这与早期文字表述的差异点在于：

- 早期描述强调的是“目标吸附到敌人组件/胸口”这一层（需要 `WasBulletParried` 或子弹时间）。
- 但“沿准星方向反弹”本身并不要求 `IsDeflecting`，`WasBulletDeflected` 路径也能做到。

已按此结论修正文档。

## 5) 脚本层：`IsBlocking` 与 `IsDeflecting` 的施加条件

下面只讨论**脚本层可见**逻辑（PSM 决策/事件），不含 native 层内部调度细节。

### 5.1 `IsBlocking`（原版脚本）

`IsBlocking` 由 `MeleeBlockEvents.OnEnter()` 施加，`OnExit()` 移除。

其进入前置由 `MeleeBlockDecisions.EnterCondition()` 决定，主要条件：

1. 玩家处于“按住格挡”状态（`IsBlockHeld`）。
2. 当前不是被招架状态（`!IsAttackParried`）。
3. 武器具备格挡能力（`CanWeaponBlock`）。
4. 非“近战瞄准+可投掷”冲突分支（`HasMeleeTargeting && CanThrowWeapon` 时拒绝）。
5. `Block` 与 `Deflect` 冷却均未激活。

满足后进入 `MeleeBlockEvents.OnEnter()`：

- 设置状态位并播放 `Block` 动画。
- 施加 `gamedataStatType.IsBlocking +1`。

退出时在 `OnExitCommon()` 中移除该 modifier。

### 5.2 `IsDeflecting`（原版脚本）

`IsDeflecting` 由 `MeleeDeflectEvents.OnEnter()` 施加，`OnExit()` 移除。

其进入前置由 `MeleeDeflectDecisions.EnterCondition()` 决定，主要条件：

1. 玩家触发“格挡按下”（`IsBlockPressed`，强调按下瞬间而非一直按住）。
2. 武器具备弹反能力（`CanWeaponDeflect`）。
3. 不能处于近战瞄准状态（`!HasMeleeTargeting`）。
4. 配置未禁用（`disabled` 参数为 false）。
5. `Deflect` 冷却未激活。

满足后进入 `MeleeDeflectEvents.OnEnter()`：

- 播放 `Deflect` 动画。
- 施加 `gamedataStatType.IsDeflecting +1`。

退出时移除 modifier，并启动 `Deflect` 冷却。

### 5.3 两者关系（脚本层结论）

- `IsBlocking` 更偏“持续防御态”（按住维持）。
- `IsDeflecting` 更偏“瞬时窗口态”（按下触发 + 持续时间/冷却控制）。
- 两者都可进入子弹防御分支，但命中标记不同：
   - `IsBlocking` 倾向 `WasBulletDeflected`
   - `IsDeflecting` 倾向 `WasBulletParried`

## 6) 与当前 Ninja 脚本的关系

你当前 `Ninja/init.lua` 中给玩家常驻了：

- `autoBlocking`：`gamedataStatType.IsBlocking +1`

这会显著提高进入“子弹防御分支”的稳定性（常态视为在挡），但**不等于**自动获得 `IsDeflecting` 的“完美窗口”。

也就是说：

- 常驻 `IsBlocking`：更容易触发 `WasBulletDeflected`（前提仍包括技能与耐力阈值）。
- 若想稳定吃到 `WasBulletParried`（完美弹反），仍需命中 `IsDeflecting` 窗口逻辑。
