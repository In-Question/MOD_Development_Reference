# DecompiledGameScripts

这些是将游戏脚本反编译为 **redscript** 的版本，便于理解函数内部实现、系统之间的交互方式，以及可用的类与函数接口。

## 目录说明

- `orphans.swift`
	- 大量枚举/常量/类型定义的汇总，便于查表与理解状态机、系统参数等。
- `core/`
	- 引擎级与通用框架层脚本（如 AI、动画、组件、黑板、系统等）。
	- `ai/`：AI 行为脚本基类、AI 行为树/条件/任务相关逻辑。
	- `components/`：通用与 AI 相关的组件实现与接口。
	- 其他子目录（`animation/`、`blackboard/`、`systems/` 等）：提供动画、黑板、系统层面的基础能力与数据定义。
- `cyberpunk/`
	- 游戏玩法层脚本（玩家、装备、任务、设备、载具、UI、战斗等）。
	- `player/`：玩家逻辑与状态机、输入响应、能力与系统交互。
	- 其他子目录（`devices/`、`quest/`、`UI/`、`vehicles/` 等）：各玩法模块的脚本实现。
- `exec/`
	- 开发/调试相关的脚本入口（如状态/调试接口、控制台相关函数）。
- `samples/`
	- 示例脚本，用于展示设备交互、动作触发等写法。
	- `ink/`：UI/Ink 相关示例。
- `tests/`
	- 测试与验证用脚本，覆盖特定交互或流程的验证场景。
	- `functionalTestsUtils/`：功能测试辅助工具。

## 用途建议

- 查找系统/类的真实实现路径与调用链
- 了解状态机、黑板、AI 行为树等底层机制
- 作为 mod 或脚本扩展的参考实现
